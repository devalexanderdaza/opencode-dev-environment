## 0. Agent Orchestration Plan

### Available agents (current context)
- `subagents/core/task-manager` (breakdown when needed)
- `subagents/code/coder-agent` (implementation)
- `subagents/code/tester` (tests, if any)
- `subagents/core/documentation` (docs)

> Constraint: avoid parallel edits to the same files. Contract files (CLI schema/state model) are single-writer tasks.

### Parallelization rules (anti-blocking)
1. **One writer per file-set**: tasks touching the same “contract surface” must be serialized.
2. **Parallelize by stream** once CLI contract + state model are stable.
3. **Docs can run in parallel** once naming and semantics are known.

### Lane mapping
- Lane A (Contract/State): `subagents/code/coder-agent`
- Lane B (Doctor/Windows ps1): `subagents/code/coder-agent`
- Lane C (Logs): `subagents/code/coder-agent`
- Lane D (Docs/Process): `subagents/core/documentation`
